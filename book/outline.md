# 하네스 엔지니어링 실전 가이드

> Claude Code의 Agent + Skill 구조를 활용한 자동화 체계 구축법

## 개요

- 대상 독자: Claude Code 사용자 중 하네스를 처음 구축하려는 개발자. CLI 기본 사용법은 알지만 에이전트 팀을 설계해본 적 없는 수준.
- 목표: 독자가 메타 스킬로 하네스를 생성하고, 생성된 구조를 이해하고 커스터마이징할 수 있게 한다.
- 총 챕터 수: 11챕터 + Ch00 (4파트)
- 산출물: 마크다운 챕터들 + 인포그래픽(SVG) + PDF + 웹 뷰어

---

## 목차

### Chapter 0: 하네스 한눈에 보기
- 목표: 하네스의 핵심 개념을 3분 안에 전달
- 핵심 내용:
  - 메타 스킬 실행 → 에이전트 팀 + 오케스트레이터 자동 생성 → 작업 시작
  - 실제 동작 예시: book-writer 하네스로 이 책이 만들어진 과정
  - 독자 수준별 추천 읽기 경로
- 분량: 4~5페이지

---

### Part 1: 핵심 개념 (Ch01~02)

#### Chapter 1: Claude Code의 Agent와 Skill 이해하기
- 목표: Agent와 Skill의 차이를 명확히 구분하고, 각각의 역할과 파일 위치를 설명할 수 있다
- 핵심 내용:
  - Claude Code의 확장 구조 개관 (`.claude/` 디렉토리)
  - Agent: "누가 하는가" — 전문가 페르소나 + 행동 원칙
  - Skill: "어떻게 하는가" — 절차적 가이드 + 도구 번들
  - 파일 위치: `.claude/agents/*.md` vs `.claude/skills/*/SKILL.md`
  - frontmatter 구조 (`name`, `description`, `model`)
  - 트리거 키워드의 역할
- 인포그래픽: Agent vs Skill 비교 다이어그램
- 분량: 10~12페이지

#### Chapter 2: 하네스와 메타 스킬
- 목표: 하네스와 메타 스킬의 관계를 이해하고, 메타 스킬 실행으로 하네스가 생성되는 전체 과정을 설명할 수 있다
- 핵심 내용:
  - 단일 에이전트의 한계와 팀 구성의 필요성
  - 하네스 = 에이전트 팀 + 오케스트레이터 스킬
  - 메타 스킬(`/harness`): 한 번의 요청으로 팀을 자동 생성
  - 메타 스킬의 6단계 워크플로우 (도메인 분석 → 검증)
  - 실전 시연: "기술 블로그 자동화" 하네스 생성 과정
  - 에이전트 분리 기준 4가지: 전문성, 병렬성, 컨텍스트 부담, 재사용성
  - 생성된 산출물 검증 체크리스트
- 인포그래픽: 메타 스킬 흐름도, 하네스 구성 요소 전체 맵
- 분량: 16~18페이지

### Part 2: 생성된 하네스의 구조 (Ch03~05)

> 메타 스킬이 자동 생성한 파일들의 구조를 이해하고, 커스터마이징하는 방법을 다룬다.

#### Chapter 3: 4가지 아키텍처 패턴
- 목표: 메타 스킬이 선택하는 4가지 패턴을 이해하고, 도메인에 맞는 패턴을 판단할 수 있다
- 핵심 내용:
  - 파이프라인, 팬아웃/팬인, 전문가풀, 생성-검증
  - 패턴 선택 의사결정 트리
  - 패턴 조합: 실전에서는 여러 패턴이 Phase별로 혼합됨
- 인포그래픽: 4가지 패턴 흐름도 + 선택 가이드
- 분량: 14~16페이지

#### Chapter 4: 에이전트·스킬·오케스트레이터의 구조
- 목표: 메타 스킬이 생성한 에이전트/스킬/오케스트레이터 파일의 구조를 이해하고, 필요시 수정할 수 있다
- 핵심 내용:
  - 에이전트 파일 구조: frontmatter + 역할 선언 + 핵심 역할 + 작업 원칙 + 출력 형식 + 협업
  - 스킬 파일 구조: frontmatter + 워크플로우 + 도구 + 출력 규칙 + references/
  - 좋은 정의 vs 나쁜 정의 비교 (커스터마이징 포인트)
  - model 선택 기준: sonnet vs opus
  - description 트리거 키워드 설계
- 분량: 14~16페이지

#### Chapter 5: 오케스트레이터와 팀 운영
- 목표: 오케스트레이터 스킬의 구조를 이해하고, Phase별 워크플로우를 읽고 수정할 수 있다
- 핵심 내용:
  - 오케스트레이터의 역할: 에이전트 간 데이터 흐름과 실행 순서 관리
  - 에이전트 구성 테이블, Phase별 워크플로우, 검증 기준
  - 파일 기반 통신: 산출물 디렉토리 구조와 네이밍 규칙
  - "각 Phase 완료 후 사용자 확인" 패턴
  - 루프 종료 조건과 에스컬레이션
- 인포그래픽: 오케스트레이터 실행 흐름도
- 분량: 12~14페이지

### Part 3: 실전 사례 (Ch06~08)

> 메타 스킬로 생성된 실제 하네스 3개를 분석한다.

#### Chapter 6: 사례 1 — 책 집필 하네스 (파이프라인 + 생성-검증)
- 목표: book-writer 하네스의 전체 구조를 이해하고, 복합 패턴 설계를 할 수 있다
- 핵심 내용:
  - 5명의 에이전트 팀 구조 분석
  - Phase 1~5 워크플로우 상세
  - 파이프라인 + 생성-검증 패턴 조합
  - 산출물 디렉토리 구조와 파일 네이밍
- 인포그래픽: book-writer 하네스 전체 아키텍처
- 분량: 16~18페이지

#### Chapter 7: 사례 2 — 리서치 팀 하네스 (팬아웃/팬인)
- 목표: 병렬 작업이 필요한 도메인에서 팬아웃/팬인 패턴을 적용할 수 있다
- 핵심 내용:
  - 병렬 에이전트 설계, 팬아웃/팬인 전략
  - 서브 에이전트 호출과 실패 처리
- 인포그래픽: 팬아웃/팬인 데이터 흐름도
- 분량: 14~16페이지

#### Chapter 8: 사례 3 — 코드 생성 하네스 (전문가풀 + 생성-검증)
- 목표: 전문가풀 패턴으로 도메인별 전문가를 라우팅하고, 생성-검증 루프를 구현할 수 있다
- 핵심 내용:
  - 라우터 역할의 오케스트레이터, 전문가별 에이전트
  - 검증 실패 시 재생성 루프
  - 전문가풀 확장 전략
- 분량: 14~16페이지

### Part 4: 운영과 확장 (Ch09~11)

#### Chapter 9: 디버깅과 개선
- 목표: 하네스 운영 중 발생하는 문제를 진단하고 개선할 수 있다
- 핵심 내용:
  - 흔한 문제 패턴: 산출물 불일치, 무한 루프, 컨텍스트 초과
  - 트리거 키워드 미스매치 디버깅
  - 에이전트 분리/통합 판단
  - 성능 최적화: model 선택, 컨텍스트 절약
- 분량: 12~14페이지

#### Chapter 10: 나만의 하네스 설계하기
- 목표: 독자 자신의 도메인에 맞는 하네스를 설계하고 구축할 수 있다
- 핵심 내용:
  - 하네스 필요성 판단 체크리스트
  - 하네스 설계 워크시트
  - 점진적 구축 전략
  - 자가 검증 체크리스트
- 인포그래픽: 하네스 설계 전체 프로세스 요약
- 분량: 14~16페이지

#### Chapter 11: 메타 스킬 커스터마이징
- 목표: 하네스 메타 스킬 자체를 수정하여 자신의 팀/조직에 맞게 확장할 수 있다
- 핵심 내용:
  - 메타 스킬 파일 구조와 위치 (`~/.claude/skills/harness/`)
  - references/ 커스터마이징: 패턴 레퍼런스, 팀 예시 수정
  - 기존 에이전트/스킬과의 충돌 방지 전략
  - 조직 표준 템플릿 만들기
- 분량: 10~12페이지
