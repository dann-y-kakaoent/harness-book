### [Ch09-W1] Phase 2 코드와 Phase 3 리뷰 피드백의 타임라인 혼란

- **위치**: Phase 2 Python 코드 (line 614)와 Phase 3 리뷰 CRITICAL 이슈 (line 928~)
- **문제**: Phase 2에서 보여주는 Python 코드의 `validate_token` 함수는 이미 `algorithms: list[str] | None = None`으로 작성되어 있다 (수정 후 버전). 그런데 Phase 3 리뷰에서 "algorithms 파라미터의 기본값이 mutable list이다"라는 CRITICAL 이슈를 제기하고, "수정 전" 코드로 `algorithms: list[str] = ["HS256"]`을 보여준다. Phase 2의 코드가 이미 수정 완료 상태라면 Phase 3의 리뷰 시나리오와 모순된다.
- **제안**: Phase 2의 코드를 의도적으로 "수정 전" 버전(`algorithms: list[str] = ["HS256"]`)으로 변경하여, Phase 3 리뷰에서 문제를 발견하고 수정하는 자연스러운 흐름을 만들거나, Phase 3 리뷰 시나리오 도입부에 "Phase 2에서 보여준 것은 최종 수정본이다. 리뷰 과정을 재현하기 위해 초안 시점의 코드를 살펴보자"라는 설명 추가

### [Ch09-W2] Java 코드의 Claim 변환 방식 기술적 정확성

- **위치**: Java TokenValidator.java (line 784~788)
- **문제**: `entry.getValue().as(Object.class)` 호출은 auth0의 java-jwt 라이브러리에서 실제로 지원되지 않는 메서드다. java-jwt의 `Claim` 인터페이스는 `asString()`, `asInt()`, `asDate()` 등의 구체적 타입 변환 메서드를 제공하지만, 범용 `as(Class<T>)` 메서드는 존재하지 않는다.
- **제안**: `entry.getValue().as(Object.class)`를 `entry.getValue().asString()`으로 변경하거나, claims를 Map으로 변환하는 별도 유틸리티를 사용하는 코드로 수정. 또는 "이 예제는 핵심 구조를 보여주기 위한 의사 코드(pseudo-code)이며, 실제 사용 시 라이브러리 API에 맞게 조정이 필요하다"라는 주석 추가

### [Ch07-S1] 에이전트 정의 코드 블록의 frontmatter 형식

- **위치**: 각 에이전트 해부 섹션 (line 292~)
- **제안**: frontmatter만 보여주고 본문을 생략한 에이전트들이 있다. infographic-creator와 book-publisher의 경우 frontmatter 이후 핵심 역할/원칙이 텍스트로만 설명되어 있어, 다른 에이전트(book-planner, chapter-writer, book-editor)처럼 코드 블록 형태로 통일하면 일관성이 높아진다.

### [Ch07-S2] 실습 Step 3의 에이전트 트리 구조

- **위치**: Step 3: 에이전트 정의 파일 작성 (line 587~)
- **제안**: 트리 구조로 에이전트 파일의 내용을 요약한 것은 좋지만, 독자가 실제로 따라하기에는 각 파일의 전체 코드가 필요하다. "앞서 분석한 에이전트 정의를 그대로 파일로 저장한다"는 한 줄을 추가하면 실습의 완성도가 높아진다.

### [Ch08-S1] 리서치 하네스의 에이전트 수 혼동

- **위치**: "이 챕터에서 배울 것" (line 7)
- **제안**: "3명의 리서처 + 1명의 분석가 + 1명의 보고서 작성자"라고 했는데, 에이전트 정의 자체는 3종류(researcher, research-analyst, report-writer)이다. 리서처 3명은 같은 에이전트를 3번 호출하는 것이므로, "3종류의 에이전트(리서처 x3 호출, 분석가, 보고서 작성자)"로 표현하면 더 정확하다. 본문에서는 이 차이를 설명하고 있으나, 도입부에서 혼동의 여지가 있다.

### [Ch08-S2] 보고서 템플릿이 실습 Step 3에 너무 상세

- **위치**: Step 3: 보고서 템플릿 (line 596~679)
